随着代理协议和网络环境的不断变化，定期更新 V2RayN 主程序及其内核是保证连接稳定性和安全性的重要手段。V2RayN 提供了非常方便的内置更新功能。

## 为什么要更新？

- **修复 Bug**：软件作者会定期修复旧版本中存在的崩溃或报错问题。
- **支持新协议**：比如最新的 Xray 核心支持 VLESS、XTLS、Reality 等新一代协议，旧版本内核可能无法解析这些节点。
- **提升性能与安全性**：更新内核可以优化加密算法，提升代理速度并降低被识别的风险。

---

## 如何在软件内自动更新

V2RayN 提供了一键更新所有相关组件的功能。

### 1. 更新 V2RayN 主程序
1. 打开 V2RayN 客户端。
2. 点击顶部菜单栏的 **“检查更新”**。
3. 选择 **“检查更新 v2rayN”**。
4. 软件会连接至 GitHub 检测最新版本，如果有更新，会自动下载并在下载完成后提示重启软件。

### 2. 更新核心组件（Xray / V2Ray / Sing-box 等）
你的节点能否顺畅连接，绝大部分取决于核心组件。
1. 在顶部菜单栏点击 **“检查更新”**。
2. 根据你当前使用的核心（默认为 Xray），选择 **“检查更新 XrayCore”**（或者 v2rayCore、sing-box Core）。
3. 等待底部日志提示下载完成。
4. **提示**：建议你把所有列出的 Core 和 Geo 规则库都更新一遍。

### 3. 更新 Geo 路由规则
路由规则文件（`geoip.dat` 和 `geosite.dat`）决定了哪些网站走代理，哪些走直连。
1. 同样在 **“检查更新”** 菜单中。
2. 点击 **“检查更新 GeoIP”** 和 **“检查更新 GeoSite”**。
3. 这能确保你拥有最新的国内外域名和 IP 数据库，使路由分流更加精准。

## 自动更新失败怎么办？（手动更新）

由于更新是直接从 GitHub 下载资源，在没有代理或当前节点失效的情况下，往往会提示“下载失败”或“连接超时”。此时你需要进行**手动更新**。

### 手动更新主程序
1. 访问 V2RayN 的 [GitHub Releases 页面](https://github.com/2dust/v2rayN/releases)。
2. 下载最新的 `v2rayN.zip`（不需要带 Core 的那个压缩包，因为你已经有核心配置了）。
3. 退出正在运行的 V2RayN 程序。
4. 将压缩包解压，把里面的文件全部覆盖到你原本的 V2RayN 文件夹中即可。

### 手动更新 Xray 内核
1. 访问 Xray-core 的 [GitHub Releases 页面](https://github.com/XTLS/Xray-core/releases)。
2. 下载最新的 Windows 64 位压缩包（通常命名为 `Xray-windows-64.zip`）。
3. 退出 V2RayN。
4. 打开 V2RayN 安装目录下的 `bin\Xray` 文件夹（如果没有就创建一个）。
5. 将下载好的压缩包解压，覆盖 `bin\Xray` 目录下的 `xray.exe` 文件。

重启 V2RayN 后，软件就会使用你手动放置的最新核心了。
