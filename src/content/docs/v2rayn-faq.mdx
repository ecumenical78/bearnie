在使用 V2RayN 的过程中，你可能会遇到无法上网或软件报错的问题。本文整理了最常见的错误场景及排查步骤。

## 1. 节点全部超时（Timeout）

如果你刚更新完订阅，或者突然发现所有节点都测速超时，无法打开网页：

### 检查系统时间
V2Ray/Xray 协议（尤其是 VMess 和 VLESS）对客户端与服务端的**时间同步要求非常严格**。误差超过 90 秒将直接导致连接失败。
**解决方法**：
1. 右键点击 Windows 任务栏右下角的时间。
2. 选择“调整日期/时间”。
3. 确保“自动设置时间”已开启，并点击“立即同步”。

### 检查防火墙与杀毒软件
某些严格的杀毒软件或 Windows Defender 可能会拦截核心程序。
**解决方法**：将 V2RayN 的所在文件夹加入杀毒软件的白名单。

### 检查网络环境
部分网络环境（如校园网、公司内网）可能封锁了代理常用的端口。你可以尝试切换节点，或联系机场客服确认服务器是否正常。

## 2. 软件报错 `Failed to start` 或闪退

此类问题通常与环境缺失或端口冲突有关。

### 端口被占用
V2RayN 默认使用 `10808` 作为本地 Socks 端口，`10809` 作为 HTTP 端口。如果这些端口被其他软件（如其他代理客户端）占用，核心将无法启动。
**解决方法**：
在 V2RayN 主界面点击 **“设置” -> “参数设置” -> “核心基础设置”**，尝试修改本地 Socks 端口为 `10810` 等不冲突的端口。

### 缺少 .NET 运行时
V2RayN 的运行依赖微软的 .NET 框架。如果系统提示缺少库文件：
**解决方法**：前往微软官网下载并安装 [**.NET Desktop Runtime 6.0** 或更高版本](https://dotnet.microsoft.com/download/dotnet)。

## 3. 开启代理后依然无法访问外网

节点测速正常（有延迟数字），但浏览器依然打不开 Google/YouTube？

### 路由设置错误
可能是由于路由规则导致所有流量都走了直连。
**解决方法**：检查底部的“路由”选项，确保选择的是 **“绕过大陆(Whitelist)”** 或 **“全局(Global)”**，而不是“直连”。

### 浏览器插件冲突
如果你在浏览器中安装了 SwitchyOmega 或其他代理插件，它们接管了浏览器的代理设置。
**解决方法**：确保浏览器插件选择的是“系统代理”，或在插件中手动配置指向 `127.0.0.1:10808`。

### DNS 污染问题
部分网页加载缓慢或无法解析，可能是本地 DNS 出现了问题。
**解决方法**：在网络适配器设置中，将系统的 DNS 修改为公共 DNS，如 `8.8.8.8` 或 `1.1.1.1`。

## 4. 出现 "TLS handshake timeout" 错误

在查看底部的日志时，如果频繁出现 `TLS handshake timeout`，这代表你的连接被阻断或者节点服务器已被墙。

**解决方法**：
1. **更换节点**：首先尝试切换到其他国家或地区的节点。
2. **更新订阅**：机场可能已经更换了被墙的 IP，请手动更新订阅获取最新的服务器节点信息。
