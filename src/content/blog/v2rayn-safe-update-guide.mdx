# 手把手教您在 2024 年更新 V2RayN：如何避免节点配置、白名单丢失？

在开源软件界，V2RayN 以其极高的迭代频率而著称。在 2024 年，它更是几乎每个月都会修复 Bug 或引入新内核的调用接口。

当软件在右下角弹出 **“发现新版本的 v2rayN”** 时，许多新手陷入了纠结：
- 点自动更新，很容易半路卡主报错 `Download Failed`。
- 如果删了旧版直接下新版，之前辛辛苦苦倒腾的自定义路由规则、以及那一长条还没到期的机场订阅，是不是就全都“灰飞烟灭”了呢？

别慌，今天我们来教大家**最稳健、零风险**的更新方法。

## 方案一：“自动更新”的避坑指北

通过 V2RayN 自带的界面更新是最简单的，但在 2024 年您必须注意以下几点：

1. 打开 V2RayN 主界面，点击上方菜单 -> **“检查更新”** -> **“检查更新v2rayN”**。
2. 此时软件会从 GitHub 获取最新版本的压缩包（注意：这个过程**要求这台电脑当前正连着海外网络**，如果 V2RayN 此时没开启接管系统代理，下载进度条会永远是 0%）。
3. 下载完成后，软件通常会提示重启关闭自身，然后解压新版本覆盖原有的 `v2rayN.exe`。

**隐患：** 大多数人在这一步遭遇失败，因为 Windows 频繁的权限管辖或是杀毒软件误杀了正在被覆盖的文件。这种时候，我们更推荐“绿色手动搬家法”。

## 方案二：手动覆盖 & 配置无损搬家法（2024 推荐 🌟）

这是我们最推荐的办法，既能享受纯净的新版，又能完美继承老数据。

### 第一步：找出这三个“命根子”文件 📁
不论 V2RayN 的大框架怎么变，您的所有机场订阅和设置数据，其实都乖乖躺在 `v2rayN` 安装目录根底下的这两个配置文件里：

*   `guiNConfig.json`：存放您的各项基础设置参数、本地监听端口等。
*   `guiNDB.db` (或旧版的 `guiNConfig-Nodes.json`)：这是真正的“命根子”。它是一个小型数据库，里面存了您的上千个节点、机场订阅地址大全。
*   `user-wininet.json` (如有)：如果您之前在高级路由里写了白名单分流，它们大多存放于此。

**操作：**
请在旧版的 `v2rayN` 文件夹中，选中上述的三个文件，右键复制，然后**粘贴备份到您的桌面上**。

### 第二步：去官方下载最新纯净版
1. 彻底退出正在运行的旧版 V2RayN（右下角托盘右击选择退出）。
2. 去 [V2RayN 的 GitHub 官方发布页](https://github.com/2dust/v2rayN/releases) 下载最新版的 `v2rayN-Core.zip`（切记下载带 Core 字样的版本，保证内核也是新鲜出炉的）。
3. 将下载好的压缩包解压到一个原本没有中文路径的新文件夹里（例如 `D:\Software\v2rayN_New`）。

### 第三步：让旧账号“附体”新软件
1. 打开刚才您放在桌面作为备份的那三个文件（`guiNConfig.json`、`guiNDB.db` 等）。
2. 把它们**拖回**我们在这个新解压出来的 `D:\Software\v2rayN_New` 文件夹里。
3. 系统提示“包含同名文件”，选择 **“替换”**！

### 第四步：启动即可
重新双击运行这个新文件夹里的 `v2rayN.exe`，您会惊奇地发现：
软件版本已经是最新版，界面排版可能更现代了，甚至底部多了几项新协议选择——**但您的名字、之前买的机场节点、分流规则、测出来的延迟数值，全都完好无损地躺在那里！**

## 总结

在 2024 年以及后续的大版本更迭中，**只要您牢牢捏住了存放订阅和参数的 `.json` 配置文件和本地的 `.db` 数据库**，您随时都可以像换衣服一样轻松切换不同版本的 V2RayN 客户端。快去把那讨厌的红点升级提示消灭掉吧！
